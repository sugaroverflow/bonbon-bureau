# Public talk notes

Bonbon Bureau is a way to explain a personal multi-agent setup without pretending every task needs a generic swarm.

## Core contrast

- **Named agents**: taste, continuity, domain judgment, trust.
- **Operator**: routing, status, escalation.
- **Candyland**: future local worker factory for chores.

## Useful framing

> Do the chores and prep the materials so my agents and I can do the interesting work.

## Demo-safe examples

- Route a product idea to Lotus.
- Route a credential rotation plan to Techie.
- Route a source monitoring task to Glyphie.
- Show a handoff packet with fake data.
